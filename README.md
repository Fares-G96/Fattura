# برنامج الفواتير العربي (InvoiceApp) — نسخة محمولة مستقلة (Portable / Self-Contained)

برنامج فواتير عربي لسطح المكتب، C# / Windows Forms، يُنشر كنسخة **محمولة مستقلة تمامًا**:
لا تثبيت، لا صلاحيات مسؤول، لا حاجة لتثبيت Excel أو .NET أو Java على جهاز المستخدم.

**واجهة البرنامج تدعم 5 لغات:** العربية (RTL افتراضية)، الإنجليزية، الفرنسية، الإيطالية،
الألمانية — تُختار من "الإعدادات ← لغة واجهة البرنامج" (يتطلب إعادة تشغيل البرنامج لتطبيقها
بالكامل). الفاتورة المطبوعة/PDF نفسها تبقى بالعربية دائمًا بغضّ النظر عن لغة الواجهة المختارة.

**البرنامج يدعم تحديثًا تلقائيًا كاملاً عبر GitHub Releases** (تحقق عند التشغيل، تحديث جزئي أو
كامل، تحقق SHA256، تراجع تلقائي عند الفشل). التفاصيل الكاملة وخطوات الإعداد في
[`UPDATER.md`](UPDATER.md) — الخطوة الوحيدة المطلوبة منك هي تعديل `InvoiceApp/config.json`
باسم مستخدم ومستودع GitHub الخاصَّين بك.

## ✅ نسخة واحدة فقط تعمل على كل الأجهزة — وليس نسخة لكل ويندوز

`publish-portable.bat` ينتج **مجلدًا واحدًا فقط** (`publish\InvoiceApp`) يحتوي على عنصرَين فقط في
مستواه الأعلى — منظَّمَين هكذا حتى لا تبحث عن ملف التشغيل وسط عشرات ملفات الـ DLL:

```
publish\InvoiceApp\
├── Fattura.exe      ← هذا فقط ما تنقر عليه لتشغيل البرنامج
└── Runtime\             ← كل ملفات DLL والبيانات (AppData) هنا، بعيدة عن الأنظار
    ├── InvoiceApp.exe
    ├── ...عشرات ملفات DLL...
    └── AppData\
```

`Fattura.exe` مُشغِّل صغير جدًا (~130 كيلوبايت فقط) مكتوب بلغة C خالصة (لا يعتمد على .NET أو أي
Runtime إطلاقًا)، بأيقونة البرنامج الخاصة به، يفتح `Runtime\InvoiceApp.exe` مباشرة وبصمت تام دون أي
نافذة سوداء أو نافذة إضافية تظهر. المسار بداخله **نسبي** (يُحسب من مكان الملف نفسه وقت التشغيل)،
فيستمر بالعمل بشكل صحيح مهما نسخت مجلد `InvoiceApp` كاملاً إلى مكان آخر (فلاشة USB، جهاز مختلف، أي
مسار) — طالما نسخته **بالكامل** مع مجلد `Runtime` المصاحب له.

هذا المجلد الواحد (`publish\InvoiceApp`) تنسخه كما هو على **أي** جهاز ويندوز: Windows 7 SP1 حتى
Windows 11، 32-بت أو 64-بت، بلا استثناء. لا يوجد "نسخة لويندوز 7" و"نسخة لويندوز 10" منفصلتين — بناء
واحد يغطي الجميع.

**كيف يعمل بناء واحد على كل الأنظمة؟** البناء 32-بت (x86)، وكل أنظمة ويندوز 64-بت تشغّل برامج 32-بت
تلقائيًا عبر طبقة توافق مدمجة في ويندوز نفسه اسمها WOW64 (تعمل منذ Windows XP 64-bit وحتى اليوم)،
فلا حاجة لبناء نسخة 64-بت منفصلة إطلاقًا.

**لماذا لم نستخدم PublishSingleFile (دمج كل شيء داخل exe واحد فعليًا)؟** جرّبناه، لكن تبيّن أنه يحمل
خطأ موثّقًا رسميًا في .NET Core 3.1 تحديدًا (تم إصلاحه لاحقًا في .NET 5+ فقط): البرنامج المنشور كملف
واحد على هذا الإصدار بالذات يخطئ في تحديد مكان نفسه الحقيقي، فيحفظ بياناته (`AppData`) داخل مجلد
مؤقت مخفي بدل المجلد المحمول بجانبه — وهذا يخالف تمامًا فكرة "برنامج محمول يُنسخ وتُنقل بياناته معه".
الحل بمجلد `Runtime` + مُشغِّل `Fattura.exe` أعلاه يعطي نفس الشعور العملي (ملف واحد تنقر عليه، بأيقونة
احترافية خاصة به) بدون هذه المشكلة، ويبقى التطبيق الفعلي `InvoiceApp.exe` (متعدد الملفات بجواره) يعمل
بشكل طبيعي وموثوق تمامًا.

## ⚠️⚠️ مهم جدًا لتشغيله على Windows 7 تحديدًا (اقرأ هذا أولًا)

النشر يجب أن يكون بمعرّف `win7-x86` وليس `win-x86` العام — هذا مُصحَّح بالفعل في `InvoiceApp.csproj`
و`publish-portable.bat` الآن.

**لكن حتى مع ذلك، أجهزة Windows 7 القديمة (خصوصًا التي لم تُحدَّث منذ فترة طويلة) غالبًا تفتقد مكوّنًا
نظاميًا واحدًا يحتاجه أي تطبيق .NET Core تقريبًا: "Universal C Runtime".** بدونه يفشل تشغيل
`InvoiceApp.exe` برسالة تحديدًا مثل:

> ‎The program can't start because api-ms-win-crt-runtime-l1-1-0.dll is missing from your computer.

هذه مشكلة **نظام تشغيل**، وليست خطأ في هذا المشروع — تحدث مع أي برنامج .NET Core/.NET 5+ مبني كنسخة
محمولة يُشغَّل على Windows 7 لم يُحدَّث. الحل بإحدى طريقتين:

**الطريقة الأسهل (تحديث النظام مرة واحدة):** ثبّت تحديث ويندوز **KB2999226** على جهاز Windows 7
(يتطلب SP1 مثبتًا مسبقًا). رابط التحميل المباشر من مايكروسوفت:
- 32-بت: https://www.microsoft.com/en-us/download/details.aspx?id=49092
- 64-بت: https://www.microsoft.com/en-us/download/details.aspx?id=49093

بعد تثبيته (وإعادة تشغيل الجهاز)، يعمل `InvoiceApp.exe` مباشرة بدون أي مشاكل، ولا تحتاج تكراره لاحقًا
لأي برنامج .NET آخر أيضًا.

**الطريقة البديلة (بدون لمس جهاز العميل نهائيًا):** انسخ ملفات الـ Universal CRT
(`ucrtbase.dll` وملفات `api-ms-win-crt-*.dll`، حوالي 40 ملفًا صغيرًا) من مجلد
`C:\Windows\System32\downlevel` على أي جهاز Windows 10 لديك، أو من حزمة "Windows 10 SDK" تحت
`Redist\ucrt\DLLs\x86`، والصقها **داخل مجلد `Runtime` بجانب `InvoiceApp.exe` مباشرة**
(`publish\InvoiceApp\Runtime\`، وليس المجلد الأعلى الذي فيه `Fattura.exe`). Windows يبحث عن
الـ DLLs في مجلد البرنامج نفسه أولًا قبل النظام، فتُستخدم هذه النسخ المحلية تلقائيًا دون حاجة
لتثبيت أي تحديث على جهاز العميل إطلاقًا. هذا التوزيع مسموح به رسميًا من مايكروسوفت تحديدًا لهذا
الغرض (توزيع محلي/Redistributable للتطبيق).

لم أستطع تضمين هذه الملفات مباشرة في هذا المشروع لأنها ملفات ثنائية من مايكروسوفت غير متاحة لي في
بيئة التطوير الحالية (لا Windows ولا اتصال بخوادم مايكروسوفت هنا) — يلزم نسخها يدويًا مرة واحدة من
جهاز Windows 10 أو من حزمة SDK إن اخترت الطريقة الثانية.

## ⚠️ نقطة هندسية مهمة يجب معرفتها

طلبت نسخة **محمولة تمامًا** (بدون تثبيت أي Runtime) **مع** دعم **Windows 7 SP1**. من الناحية التقنية،
هذان الشرطان معًا يحصران الخيار في منصة واحدة فقط:

- **.NET Framework 4.8** (المستخدم في الإصدار الأول من هذا المشروع): لا يمكن توزيعه كنسخة محمولة —
  إنه جزء من نظام التشغيل نفسه، ويتطلب تثبيتًا منفصلاً (ويتطلب صلاحيات مسؤول) على أي جهاز Windows 7
  لا يملكه مسبقًا. هذا يتعارض مباشرة مع شرط "بدون تثبيت / بدون صلاحيات مسؤول".
- **.NET 5 وما بعده** (.NET 6, 7, 8...): يدعم النشر المحمول المستقل (Self-Contained) بشكل ممتاز،
  **لكنه أسقط دعم Windows 7 رسميًا** (الحد الأدنى المدعوم هو Windows 10). هذا يتعارض مع شرط دعم
  Windows 7 SP1.
- **.NET Core 3.1**: هو **الإصدار الوحيد** الذي يجمع بين الأمرين معًا — يدعم Windows 7 SP1 رسميًا،
  **و** يدعم النشر المستقل (Self-Contained) الذي يحزم كل شيء داخل مجلد البرنامج بدون أي تثبيت.

لذلك تم ترحيل المشروع بالكامل من .NET Framework 4.8 إلى **.NET Core 3.1** لتحقيق طلبك حرفيًا.

**السلبية الوحيدة الواجب معرفتها:** دعم .NET Core 3.1 الرسمي من مايكروسوفت (تحديثات أمنية) **انتهى في
13 ديسمبر 2022**. البرنامج سيعمل تمامًا بنفس الكفاءة، لكنه لن يتلقى تحديثات أمنية لمكوّنات الـ Runtime
نفسها مستقبلاً. إن لم يكن دعم Windows 7 SP1 شرطًا حتميًا لديك، فالبديل الأفضل أمنيًا هو استخدام .NET 8
(LTS مدعوم حاليًا) والتخلي عن دعم Windows 7 فقط — أخبرني إن كنت تفضّل هذا المسار بدلاً من ذلك.

## ما تغيّر تقنيًا لتحقيق الاستقلالية الكاملة

1. **الانتقال إلى .NET Core 3.1** (بدل .NET Framework 4.8) — يسمح بنشر Self-Contained.
2. **حذف مكتبة iTextSharp نهائيًا.** كانت تُستخدم لتوليد PDF، لكن الاعتماد عليها كان يُعقّد ضمان توافقها
   الكامل مع النشر المستقل. بدلاً منها، بُنيت طبقة PDF ذاتية بالكامل:
   - الفاتورة تُرسم عبر `System.Drawing` (GDI+) القياسي في ويندوز — وهو **جزء من نظام التشغيل نفسه**
     في كل نسخ Windows المستهدفة، ويُشكّل الحروف العربية تلقائيًا وبشكل صحيح دون أي مكتبة إضافية.
   - نفس منطق الرسم يُستخدم لكل من "طباعة" و"تصدير PDF" (ملف `Reports/InvoiceRenderer.cs`) لضمان
     تطابق الشكل بينهما 100%.
   - ملف PDF نفسه يُكتب يدويًا بكود بسيط (`Reports/SimplePdfWriter.cs`) يُضمّن كل صفحة كصورة JPEG
     داخل هيكل PDF صالح — **بدون أي مكتبة PDF خارجية على الإطلاق**.
3. **مكتبتا NuGet الوحيدتان المتبقيتان:** `ClosedXML` (قراءة إكسل) و`Newtonsoft.Json` (حفظ الإعدادات
   والفواتير)، وكلتاهما Managed بالكامل ومتوافقتان مع النشر المستقل دون أي مشاكل.
4. **مجلد البيانات أصبح محمولاً:** الإعدادات والفواتير تُحفظ الآن داخل مجلد `AppData` **بجانب
   InvoiceApp.exe مباشرة** (وليس في %AppData% الخاص بالمستخدم كما في النسخة السابقة)، بحيث يمكن نسخ
   مجلد البرنامج بالكامل (بما فيه بيانات الفواتير) إلى جهاز آخر أو USB والاستمرار من حيث توقفت.

### عن `Launcher/Fattura.exe`

ملف تنفيذي صغير (~130 كيلوبايت) مكتوب بلغة C خالصة عبر MinGW، **لا يعتمد على .NET أو أي Runtime
إطلاقًا** — لذلك يعمل فورًا على أي إصدار ويندوز من 7 إلى 11 بلا أي شرط مسبق. مهمته الوحيدة: إيجاد
`Runtime\InvoiceApp.exe` بجانبه وتشغيله بصمت. **هذا الملف مبنيّ مسبقًا وجاهز — لا داعي لإعادة بنائه
عند أي تعديل على المشروع نفسه**، لأنه لا يحتوي على أي منطق خاص بالفاتورة، فقط يبحث عن ويشغّل الملف
الحقيقي بجانبه بالاسم الثابت `Runtime\InvoiceApp.exe`. يُعاد بناؤه فقط لو أردت تغيير الأيقونة أو اسم
المُشغِّل نفسه مستقبلاً (يتطلب MinGW، غير متاح افتراضيًا على معظم أجهزة ويندوز — التفاصيل غير مطلوبة
للاستخدام العادي).

## متطلبات جهاز **التطوير** فقط (وليس جهاز المستخدم النهائي)

- Visual Studio 2019/2022، أو .NET SDK 3.1 من سطر الأوامر فقط.
- تنزيل .NET SDK 3.1.x من: `https://dotnet.microsoft.com/download/dotnet/3.1` (اختر SDK وليس Runtime).
- اتصال إنترنت لمرة واحدة لاستعادة حزمتي NuGet (ClosedXML, Newtonsoft.Json) وحزمة الـ Runtime الخاصة
  بالنشر المستقل.

جهاز **المستخدم النهائي** لا يحتاج أي مما سبق إطلاقًا.

## بناء النسخة المحمولة (الطريقة الموصى بها)

من داخل مجلد الحل (بجانب `InvoiceApp.sln`)، شغّل:

```
publish-portable.bat
```

سينشئ المجلد `publish\InvoiceApp\` بالبنية الموضحة أعلى الملف (`Fattura.exe` في الأعلى، وكل شيء
آخر داخل `Runtime\`). **انسخ مجلد `publish\InvoiceApp` بالكامل** (وليس ملف واحد فقط منه) إلى أي جهاز
هدف — **هو نفسه المجلد الذي تنسخه لكل الأجهزة**، سواء Windows 7 SP1 أو Windows 10 أو 11، 32-بت أو
64-بت، بلا أي استثناء أو نسخة بديلة.

### بناء يدوي بديل (سطر أوامر مباشر)

```
dotnet publish InvoiceApp\InvoiceApp.csproj -c Release -r win7-x86 --self-contained true -o publish\InvoiceApp\Runtime
```

هذا الأمر وحده ينشر الملفات داخل `Runtime\` فقط، دون وضع `Fattura.exe` تلقائيًا (تلك خطوة إضافية
داخل `publish-portable.bat`). إن استخدمت هذا الأمر يدويًا، إما شغّل `Runtime\InvoiceApp.exe` مباشرة،
أو انسخ ملف `Launcher\Fattura.exe` الجاهز مسبقًا (مبني مرة واحدة، ثابت، لا يحتاج إعادة بناء) إلى
جانب مجلد `Runtime`.

### من Visual Studio

انقر بزر الفأرة الأيمن على مشروع InvoiceApp ← Publish ← Folder ← اختر مجلد الوجهة
`publish\InvoiceApp\Runtime` تحديدًا (وليس `publish\InvoiceApp` نفسه) —:
- Target Runtime: `win7-x86`
- Deployment Mode: `Self-contained`
- Produce single file: **إيقاف** (Off)

## اختبار على جهاز بدون Excel وبدون .NET (كما طلبت)

1. انسخ مجلد `publish\InvoiceApp` بالكامل (عبر فلاشة USB مثلاً) إلى جهاز نظيف — يفضّل جهاز افتراضي
   (Virtual Machine) بدون أي برامج مثبتة، أو جهاز Windows 7 SP1 فعلي لم يُثبَّت عليه .NET Framework 4.8
   ولا Microsoft Office.
2. شغّل `Fattura.exe` مباشرة بالنقر المزدوج (وليس الملف داخل `Runtime`) — يجب أن يفتح البرنامج
   فورًا دون أي رسالة تطلب تثبيت شيء ودون طلب صلاحيات مسؤول (لا تظهر نافذة UAC)، ودون ظهور أي نافذة
   سوداء (Console).
3. من زر "الإعدادات"، اختر ملف `Book1.xlsx` (يمكنك نسخه من مجلد `SampleData` المرفق) — لاحظ أن قراءته
   لا تتطلب وجود Microsoft Excel على الجهاز إطلاقًا (`ClosedXML` تقرأ ملف xlsx مباشرة كأرشيف).
4. جرّب إضافة صنف، الحفظ، ثم أغلق البرنامج وأعد فتحه — يجب أن تجد الفاتورة محفوظة داخل مجلد
   `publish\InvoiceApp\Runtime\AppData` (بجانب `InvoiceApp.exe` الفعلي داخل مجلد Runtime).
5. جرّب "تصدير PDF" وتأكد من ظهور الفاتورة بالعربية من اليمين لليسار بشكل صحيح.
